## 2. GTMフェーズ

### Phase 1: ビルド可能にする（P0 — 今週中）

**ゴール**: macOS上でビルド→署名→公証→DMGが一通り動く

- [ ] Xcodeで全ソースのコンパイルエラーを解消
- [ ] Apple Developer Program加入確認（Developer ID Application証明書）
- [ ] scripts/build.sh: xcodebuild archiveコマンド追加
- [ ] scripts/notarize.sh: 環境変数設定、テスト実行
- [ ] 手動でDMG作成→インストール→Gatekeeperを通過確認
- [ ] .github/workflows/release.yml: TODO部分のアンコメント・修正

### Phase 2: 配布可能にする（P1 — 来週中）

**ゴール**: 第三者がダウンロードして使える状態

- [ ] GitHub Releasesに v0.1.0-beta をアップロード
- [ ] README.mdにダウンロードリンク・インストール手順を追記
- [ ] LP作成（最低限: スクショ + DLリンク + プライバシーポリシー）
- [ ] Sparkle統合（appcast.xml生成、SUUpdater初期化）
- [ ] フィードバック導線（GitHub Issues or フォーム）

### Phase 3: 課金可能にする（P1 — 2週間以内）

**ゴール**: Pro版の購入→アンロックが動く

- [ ] Stripe アカウント + 商品設定
- [ ] ライセンスキー生成ロジック（Stripe Webhook → キー発行）
- [ ] アプリ内ライセンスキー入力UI + 検証ロジック
- [ ] hardPaywallChoice画面の「Proにアップグレード」ボタンをStripeに接続
- [ ] softPaywall画面のCTAも同様に接続

### Phase 4: 初期ユーザー獲得

**ゴール**: 最初の100人のユーザー

- [ ] Xでの告知（開発過程の共有）
- [ ] Product Hunt投稿
- [ ] Hacker Newsへの投稿
- [ ] 知人への直接共有（5-10人のクローズドベータ）

## 3. 価格設計

| 項目 | 値 |
|------|-----|
| 無料版 | 基本セッション（5秒閉眼）、30分固定間隔 |
| Pro版 | 買い切り（価格未定、$9-19想定） |
| Pro追加機能 | Pre-close/Post-close、週次レポート、暴走検知、カスタマイズ |

**注意**: 現在のコードでは無料版でも診断→パーソナライズ間隔が使える設計。Pro機能の境界を明確にする必要がある。

### 無料版 vs Pro版の機能境界（提案）

| 機能 | 無料版 | Pro版 |
|------|--------|-------|
| 基本セッション | o | o |
| 拡張オンボーディング（診断） | o | o |
| パーソナライズ間隔 | o | o |
| トレーニング時間帯 | o | o |
| 5分猶予 | o | o |
| Pre-close（内なる声チェック） | - | o |
| Post-close（象使いの判断） | - | o |
| 暴走エスカレーション | 通知テキストのみ | 間隔短縮+テキスト |
| Day 7レポート | 概要のみ | 詳細+ヒートマップ |
| 週次レポート（継続） | - | o |
| カレンダー連携 | - | o |

## 4. リスク

| リスク | 影響 | 対策 |
|--------|------|------|
| Xcodeビルドエラー多数 | Phase 1ブロック | 早期にローカルビルド実施 |
| Apple Developer審査遅延 | Phase 1ブロック | 早めに申請 |
| Sparkle統合の複雑さ | Phase 2遅延 | MVP時は手動更新でも可 |
| Stripeの本番審査 | Phase 3遅延 | テストモードで先に実装 |
| 初期ユーザーのフィードバック過多 | 開発リソース圧迫 | クローズドベータで段階的に |
