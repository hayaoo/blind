# Blind — 拡張オンボーディング設計（顧客育成型）

## 採用方針

**パターンA（自己診断型）をベースに、パターンB（7日間チャレンジ）のDay 7成果レポート+ペイウォールを組み合わせる。**

パターンCのストーリー型は不採用（効果が不確実）。

---

## ターゲットペルソナの再定義

### 「集中が足りない人」ではなく「過集中で暴走する人」

Blindのメインターゲットは**集中力が低い人ではない**。
むしろ**集中しすぎて、象（System 1 / 無意識の衝動）が暴走し、使い（理性 / メタ認知）が手綱を失う人**。

#### 象と象使いのメタファー

```
象使い（理性）: 「今日はAタスクが最重要だ」
象（衝動）:     「お、このバグ面白い。直そう。あ、リファクタもしよう。
                 ついでにCI設定も...」

── 3時間後 ──

象使い:         「...あれ、Aタスク一切手をつけてない」
```

**Blindの5秒は、暴走した象を止めて象使いに手綱を返す瞬間。**

#### このペルソナの特徴

| 特徴 | 集中力不足の人 | **過集中の人（Blindのターゲット）** |
|------|-------------|--------------------------------|
| 問題 | 集中できない | 集中しすぎて方向を見失う |
| 時間感覚 | 「やる気が出ない」 | 「気づいたら3時間経ってた」 |
| タスク管理 | そもそも着手できない | 着手したら止まれない。抜け漏れが出る |
| AI利用時 | AI使えば楽になる | AIが象を加速させる（ウサギの穴が深くなる） |
| 休憩の障壁 | やる気がない | **没頭していて気づかない / 中断が怖い** |
| 必要なもの | モチベーション | **強制的な中断 + メタ認知の瞬間** |
| Blindの価値 | リラックス | **「今、正しいことをしているか？」の確認** |

#### PQ/EQの視点で見た過集中ペルソナ

**PQ（Positive Intelligence）**: Shirzad Chamineの理論では、人の思考を乗っ取る「サボター（内なる妨害者）」が10種類ある。過集中を駆動するサボターは主に4つ:

| サボター | 内なる声 | 暴走パターン |
|---------|---------|------------|
| **Hyper-Achiever（達成者）** | 「もっと成果を出さないと自分に価値がない」 | 次々とタスクを追加し、止まると不安になる |
| **Stickler（完璧主義）** | 「まだ完璧じゃない。もう少しだけ」 | 一つのタスクに固執して完璧を追求し続ける |
| **Restless（落ち着きなし）** | 「これも面白い、あれも面白い」 | 新しい刺激に次々飛びつき、本来のタスクから脱線 |
| **Controller（支配者）** | 「自分がやらないと回らない」 | 委譲できず全部抱え込み、暴走的に処理しようとする |

**象を暴走させているのはサボター。Blindの5秒はPQ Repそのもの。**

**EQ（感情的知性）**: Daniel Golemanの理論では、感情的知性の5要素のうち、過集中の文脈で重要なのは:

| EQ要素 | 過集中との関係 | Blindの役割 |
|--------|-------------|------------|
| **自己認知（Self-Awareness）** | 「今、暴走している」と気づけるか | 気づきのきっかけを強制的に提供 |
| **自己制御（Self-Regulation）** | 気づいた後に止められるか | 止める力を外部化（Proの段階的介入） |
| **衝動制御（Impulse Control）** | 「あと少しだけ...」を止められるか | 5秒の中断で衝動の連鎖を切る |

**核心的な問い: 「自己認知」と「自己制御」の間にギャップがある人ほど、Blindの価値が高い。**
（「暴走していると気づいているのに止められない」人 = 最も課金しやすい層）

#### Block C: 自己認知と自己制御（4問）— EQ（感情的知性）の視点

> このブロックの裏側: [Daniel GolemanのEQ理論](https://www.helpguide.org/mental-health/wellbeing/emotional-intelligence-eq)を応用。
> 感情的知性の5要素のうち、**自己認知（暴走に気づく力）**と**自己制御（気づいた後に止める力）**を測定。
> この2つの間のギャップが大きい人ほど、Blind Proの価値が高い。

| # | 質問 | 選択肢例 | EQ要素 | **Pro機能への伏線** |
|---|------|---------|--------|-------------------|
| 12 | 「"今、自分は過集中状態だ"と、最中に気づくことはありますか？」 | よく気づく / たまに気づく / 終わってから気づく / 人に指摘されて気づく | **自己認知（Self-Awareness）** | → 暴走検知の必要性レベル |
| 13 | 「気づいたとして、実際に行動を変えられますか？」 | すぐ変えられる / わかっていてもやめられない / 変えようとするが結局戻る / 気づいても無視する | **自己制御（Self-Regulation）** | → 介入の強度設計 |
| 14 | 「"今、正しいことをしているか？"を定期的に確認する習慣はありますか？」 | ある / 作りたいが続かない / 考えたこともなかった | **メタ認知習慣の有無** | → Blindの核心価値への接続 |
| 15 | 「Blindに一番期待することは？」 | 暴走を止めてほしい / 優先順位を思い出したい / 定期的に立ち止まりたい / 目を休めたい | **ゴール設定** | → メッセージ・機能の全体パーソナライズ |

**EQギャップの分類 — ユーザータイプとPro訴求の最適化**:

| タイプ | Q12（気づく） | Q13（止める） | ギャップ | Blindの価値 | Pro訴求の切り口 |
|--------|-------------|-------------|---------|------------|----------------|
| **無自覚・無制御** | 終わってから / 人に指摘 | — | 気づきすらない | 気づきの提供 + 強制中断 | 「暴走検知」最優先。気づく機会自体を作る |
| **気づくが止まれない** | よく / たまに | やめられない / 結局戻る | **最大のギャップ** | 知ってるのに止まれない → 外部制御 | 「段階的割り込み」。**最もCVRが高い層** |
| **気づいて止まれる** | よく気づく | すぐ変えられる | ギャップ小 | きっかけだけで十分 | 「優先事項アンカー」「カスタム問いかけ」。軽い介入 |
| **気づくが無視する** | よく気づく | 気づいても無視 | 意図的な放置 | 無視を許さない介入 | 「暴走検知」最強モード。あえて鬱陶しく |

#### 診断の全体設計意図

```
Block A（行動）: 何が起きているか → 「確かに暴走している」
     ↓
Block B（心理）: なぜ起きるのか → 「こういう思考パターンが原因だったのか」
     ↓
Block C（能力）: 自分で止められるか → 「気づいてるのに止められない。外部の仕組みが要る」
     ↓
結論: 「Blindが必要だ」
```

- **Block Aだけ**だと「自分は暴走する」で終わる。原因がわからない
- **Block Bを加える**と「サボターが原因だ」と内省が深まる。課題感が倍増する
- **Block Cを加える**と「原因を知っても自分では止まれない」という決定的な気づきに至る
- **3ブロックの通過自体がサンクコスト**になり、「ここまで答えたのだから使おう」という心理が働く

## Pro機能設計: 「暴走する象を止める」ための道具

### 設計原則

> **過集中ペルソナの課題から逆算する。「あったらいい」ではなく「象を止めるために必要」な機能だけ。**

従来のPro機能（見た目・音のカスタマイズ）はこのペルソナには刺さらない。
彼らが金を払うのは**「暴走の検知と介入の精度」**に対して。

---

### Pro機能①: 暴走検知リマインド（Runaway Detection）

**診断での伏線**: Q5「最長何時間連続？」/ Q10「止めるまでどのくらい？」/ Q13「気づいても止まれない」

| 項目 | 内容 |
|------|------|
| 概要 | スキップ行動から「今、暴走中」を検知し、リマインドの強度を段階的に上げる |
| 無料版 | 固定間隔（15/30/45/60分）で均一に通知 |
| Pro版 | スキップすると次の通知が短い間隔で来る。2回連続スキップ → 半分の間隔に。3回連続 → さらに短く+音の変化 |
| 技術実装 | スキップ回数カウント + 間隔の動的調整。ローカル完結 |
| ペイウォールでの訴求 | 「あなたは7日間で○回連続スキップしました。それはサボターが暴走しているサインです。Proなら見逃しません」 |

**EQの視点で見た価値**: Q13で「わかっていてもやめられない」と答えた人 = **自己認知はあるが自己制御が足りない人**。この人にとって暴走検知は「自分にない自己制御力を外部に持ってもらう」こと。

**段階的割り込みの設計**:
```
通常: 30分間隔の通知
  ↓ スキップ（サボターが活動開始）
15分後: やや強い通知（音の変化）
  ↓ またスキップ（サボターが本格稼働）
7分後: 強い通知（画面にも表示）
  ↓ またスキップ（完全に暴走）
3分後: 最強通知（ノッチUIが自動展開）
```

### Pro機能②: 優先事項アンカー（Priority Anchor）

**診断での伏線**: Q4「予定と違う作業をしていた」/ Q6「本来のタスクが漏れた」

| 項目 | 内容 |
|------|------|
| 概要 | 1日の始めに「今日やるべきこと」を1-3個設定。各セッションの閉眼前に表示 |
| 無料版 | 固定メッセージ「目を閉じて、今を確かめよう」 |
| Pro版 | 「目を閉じる前に——今日の優先事項: ①Aタスク ②Bタスク ③Cタスク」→ 閉眼 → 開眼後「今やっていたことは、この中にありましたか？」 |
| 技術実装 | メニューバーから毎朝設定（3つまで）。セッションUIに表示 |
| ペイウォールでの訴求 | 「あなたは"優先順位を思い出したい"と答えました。Proなら毎回のセッションがチェックポイントになります」 |

**PQの視点で見た価値**: サボターは「今やっていることが重要だ」と錯覚させる。優先事項アンカーは、象使いの地図を毎回見せることで、サボターの錯覚を打ち破る。

### Pro機能③: サボター別カスタム問いかけ（Saboteur-Aware Prompts）

**診断での伏線**: Q8「内なる声」/ Q9「暴走後の感情」/ Q15「期待すること」

| 項目 | 内容 |
|------|------|
| 概要 | 診断で推定したサボタータイプに基づき、セッション中のメッセージを自動パーソナライズ |
| 無料版 | 固定メッセージ（「目を閉じて、今を確かめよう」/「おかえり」） |
| Pro版 | サボター別メッセージ + ゴール別メッセージ + 完全カスタム入力 |
| ペイウォールでの訴求 | 「あなたの"内なる声"は『○○タイプ』。Proならそのサボターに最適な問いかけで介入します」 |

#### サボター別メッセージセット

| サボター | 開始メッセージ（サボターへの問い） | 閉眼中の問い | 終了メッセージ |
|---------|--------------------------------|------------|--------------|
| **Hyper-Achiever** | 「一度、手を止めてください。"もっとやれる"の声が聞こえていませんか？」 | 「十分やりました。次にやるべき"最重要"は？」 | 「"もっと"ではなく"正しいこと"を」 |
| **Stickler** | 「完璧を追わなくていい瞬間です」 | 「今の作業は80%で十分ではありませんか？」 | 「不完全でも前に進むことに価値がある」 |
| **Restless** | 「新しい刺激を追いかけていませんか？」 | 「今日のゴールに戻りましょう。面白いことは後で」 | 「一つに集中する勇気を」 |
| **Controller** | 「全部を自分でやる必要はありません」 | 「今、本当にあなたがやるべきことは何ですか？」 | 「手放す勇気も大事」 |

#### ゴール別メッセージセット（Q15回答ベース）

| ユーザーの期待 | 開始メッセージ | 閉眼中の問い | 終了メッセージ |
|--------------|--------------|------------|--------------|
| 暴走を止めてほしい | 「一度、手を止めてください」 | 「今やっていることは、始めようと思ったことですか？」 | 「手綱を取り戻しましたか？」 |
| 優先順位を思い出したい | 「今日の最重要タスクは何でしたか？」 | 「それに向かっていますか？」 | 「よし、正しい山を登ろう」 |
| 定期的に立ち止まりたい | 「立ち止まる時間です」 | 「今の自分を俯瞰してみてください」 | 「また30分後に」 |
| 目を休めたい | 「目を休めましょう」 | — | 「お疲れさまでした」 |
| 完全カスタム | ユーザーが自由に入力 | ユーザーが自由に入力 | ユーザーが自由に入力 |

**Proでは「サボター別 × ゴール別」の掛け合わせが可能。** 例: Stickler + 暴走を止めてほしい → 「"完璧じゃなきゃ"の声が聞こえていませんか？80%で次に行きましょう」

### Pro機能④: 暴走パターン分析（Runaway Insights）

**診断での伏線**: Q4「予定と違う作業をしていた」の頻度 / Day 7レポートの体験

| 項目 | 内容 |
|------|------|
| 概要 | いつ・どのくらい暴走しやすいかを可視化する週次レポート。サボターの活動パターンを追跡 |
| 無料版 | メニューバーに「今日○回」のみ表示 |
| Pro版 | 週次レポート: 曜日×時間帯のスキップ率ヒートマップ / サボター活動スコア / 連続スキップ発生回数 / 前週比較 |
| 技術実装 | セッションログ（時刻, 実行/スキップ, 連続スキップ数）をローカルDB保存 |
| ペイウォールでの訴求 | 「あなたは火曜の14-17時にスキップ率が80%——サボターが最も暴走しやすい時間帯です。Proなら毎週追跡できます」 |

**EQの視点**: ウィークリーインサイトは**自己認知の定期トレーニング**。「暴走しやすい時間帯を知っている」こと自体が自己認知力を高め、長期的にはBlindなしでも止まれるようになるかもしれない。これはユーザーの成長であり、Blindの究極の価値。

### Pro機能⑤: スマートスケジュール（Smart Schedule）

**診断での伏線**: Day 7レポートの暴走パターン

| 項目 | 内容 |
|------|------|
| 概要 | 暴走しやすい時間帯にリマインド頻度を自動で上げる |
| 無料版 | 常時均一の固定間隔 |
| Pro版 | 曜日×時間帯でリマインド間隔を変動。「火曜午後は20分間隔、金曜午前は45分間隔」 |
| 技術実装 | スキップ率の時間帯別集計 → 間隔の自動調整 |
| ペイウォールでの訴求 | 「あなたは午後にサボターが活発化する傾向があります。Proなら午後だけ間隔を短くできます」 |

## Pro機能が刺さらないもの（不採用 or 優先度下げ）

| 機能 | なぜ過集中ペルソナに刺さらないか |
|------|-------------------------------|
| ビジュアルカスタマイズ（目キャラの色変更） | 暴走を止めてくれない。「かわいい」は購買動機にならない |
| サウンドカスタマイズ | 音の種類が増えても暴走は止まらない |
| 30日/90日チャレンジ | 過集中の人は「チャレンジ達成」より「今日の暴走を防ぐ」に関心。長期プログラムは離脱しやすい |
| カレンダー連携（会議中スキップ） | 過集中の人は会議中に暴走しない。暴走するのは一人で作業している時 |

**結論**: 過集中ペルソナが金を払うのは「見た目」でも「達成感」でもなく、**「サボターの検知と介入の精度」**。

## Pro機能の価値マトリクス: 無料 vs Pro

| 領域 | 無料版 | Pro版 | 過集中ペルソナの「壁」心理 |
|------|--------|-------|------------------------|
| リマインド | 固定間隔で均一 | スキップ連続で間隔短縮（暴走検知） | 「通知を無視した時こそ必要なのに」 |
| セッションメッセージ | 固定「目を閉じて、今を確かめよう」 | サボター別 × ゴール別の問いかけ | 「"今を確かめよう"じゃサボターは止まらない」 |
| 優先事項 | なし | 毎朝設定→毎セッション表示 | 「何をすべきか知ってる。忘れるのが問題」 |
| 振り返り | 「今日○回」のみ | 暴走パターン + サボター活動分析 | 「いつ暴走するか、何が原因か知りたい」 |
| 時間帯調整 | 均一 | 暴走しやすい時間帯は高頻度 | 「午後が危ないのに朝と同じ間隔は無駄」 |

## なぜこの組み合わせが過集中ペルソナに刺さるか

### 1. 3層の診断が「自分を理解された」感覚を作る

```
Block A: 「確かに暴走している」 — 行動の事実
Block B: 「"もっとやれる"のサボターが原因だった」 — 心理的原因
Block C: 「気づいてるのに止まれないのは仕組みの問題」 — 能力のギャップ
```

一般的なアプリの診断は「スクリーンタイムは？」「瞑想経験は？」。表面的。
Blindの診断は**行動→心理→能力の3層で掘り下げる**。だから「このアプリは自分を理解している」と感じる。

### 2. サボタータイプが「自分ごと化」のエンジンになる

「あなたはHyper-Achieverタイプです」と直接言わない（押し付けがましい）。
代わりに「"もっと良くできるはず"という声が聞こえることが多いようですね」と伝える。

この「あ、それ自分だ」の瞬間が、アプリへの信頼と「Pro使ってみたい」を同時に生む。

### 3. EQギャップが「Proの必要性」を論理的に証明する

```
自己認知: 暴走に気づく力 → Blindの無料版で提供（リマインド）
自己制御: 暴走を止める力 → 自分にない → Proが外部化する

「気づきは無料で提供します。でも止める力はProが必要です」
```

これは「機能を売る」のではなく「能力の不足を補う」提案。だから押し売りではなく支援になる。

### 4. 競合にこの切り口がない

| アプリ | 提供するもの | 過集中への対応 | サボター理解 |
|--------|------------|-------------|------------|
| ポモドーロ系 | 25分固定タイマー | 均一。暴走中でも同じ音 | なし |
| Headspace | ガイド瞑想 | 瞑想する時間に過集中の人は来ない | なし |
| Stretchly | ストレッチ通知 | 均一。無視したら次は同じ間隔後 | なし |
| **Blind Pro** | **暴走検知 + サボター別介入** | **スキップするほど強く介入** | **診断で特定→メッセージ最適化** |

## リスクと対策

| リスク | 対策 |
|--------|------|
| 12問の診断で離脱率が高い | 各ブロックの冒頭にスキップ導線。Block Aのみでも最低限のプラン生成が可能 |
| 「暴走検知」が鬱陶しく感じる | Pro設定で強度を3段階に調整可能。最初はマイルドから |
| サボター分類が的外れ | 4タイプは大まかな傾向。「しっくりこない場合はカスタム入力で上書き」を用意 |
| 「象と象使い」「サボター」の用語がピンとこない | アプリ内ではPQ/EQ用語を使わず、「内なる声」「気づきと行動のギャップ」など平易な言葉で表現 |
| macOSアプリとして長いオンボーディングが異質 | ノッチUIの中で展開。モバイルのフルスクリーン感とは異なる体験として設計 |
| Day 7まで到達しない | Day 3, Day 5でもライトな成果表示。「サボターが○回検出されました」で興味を維持 |
