# Blind — 拡張オンボーディング設計（顧客育成型）

## 採用方針

**パターンA（自己診断型）をベースに、パターンB（7日間チャレンジ）のDay 7成果レポート+ペイウォールを組み合わせる。**

パターンCのストーリー型は不採用（効果が不確実）。

---

## 全体ジャーニー

```
Day 1（初回起動）:
  [導入] → [診断] → [知識] → [体験] → [処方] → [ソフトペイウォール①] → [開始]
   3画面    8画面    5画面    3画面    4画面        1画面              1画面
                                                              合計: 25画面

Day 2-6:
  デイリーTips通知（1画面/日）
  → 振り返り + 知識 + ミニ体験

Day 7:
  [成果レポート] → [ハードペイウォール②]
      3画面              2画面
```

**2段階ペイウォール戦略**:
- Day 1: ソフトペイウォール（プラン提示、軽く紹介。スキップしやすい）
- Day 7: ハードペイウォール（7日間のデータを使った説得力のある提案）

## Pro機能設計: 診断ジャーニーから逆算した価値提供

### 設計原則

> **診断で課題を自覚させ、7日間で体験させ、Proでしか解決できない形にする。**

従来のPro機能（見た目・音のカスタマイズ）は「あったらいい」レベル。
診断型オンボーディングでは、**診断で見つけた課題の解決手段**としてPro機能を位置づける。

---

### Pro機能①: 適応リマインド（Adaptive Reminders）

**診断での伏線**: Q5「何分おきに意識が逸れますか？」/ Q8「休憩を取れない理由は？」

| 項目 | 内容 |
|------|------|
| 概要 | 固定間隔ではなく、使用パターンに基づいてリマインド間隔を自動調整 |
| 無料版 | 15 / 30 / 45 / 60分の固定間隔 |
| Pro版 | 7日間のデータから「あなたは26分で集中が切れる傾向があります」→ 自動で26分間隔に最適化 |
| 技術実装 | セッション実行時刻・スキップ率・曜日/時間帯パターンからローカルで算出 |
| ペイウォールでの訴求 | 「あなたの7日間のデータから、最適な間隔は○分です。Proなら自動で調整します」 |

**なぜ刺さるか**: 診断で「自分は○分で集中が切れる」と自覚した人に、「固定間隔では合わない」→「あなた専用のリズム」は強い。7日間のリアルデータがあるので説得力が段違い。

---

### Pro機能②: ウィークリーインサイト（Weekly Insights）

**診断での伏線**: Q4「スクリーンタイムは？」/ Q7「こんな経験は？」

| 項目 | 内容 |
|------|------|
| 概要 | 1週間ごとの振り返りレポート。セッションデータの可視化と傾向分析 |
| 無料版 | メニューバーに「今日○回」のみ表示 |
| Pro版 | 週次レポート: 曜日別セッション率 / 時間帯ヒートマップ / 連続記録 / 前週比 |
| 技術実装 | CoreData or SQLiteにセッションログ。週次集計をローカルで算出 |
| ペイウォールでの訴求 | Day 7レポートをそのまま「こういうレポートが毎週届きます」として見せる |

**なぜ刺さるか**: Day 7のレポートで「自分の傾向」を初めて見た直後。「これが毎週見られる」は即座に価値が伝わる。**Day 7レポート自体がProのデモになっている**。

---

### Pro機能③: カスタムメッセージ（Custom Messages）

**診断での伏線**: Q7「こんな経験は？」/ Q10「Blindに期待することは？」

| 項目 | 内容 |
|------|------|
| 概要 | セッション開始時・終了時のメッセージを、診断結果に基づいてパーソナライズ |
| 無料版 | 固定メッセージ（「目を閉じて、今を確かめよう」/「おかえり」） |
| Pro版 | ゴール別メッセージセット + 完全カスタム入力 |
| メッセージ例 | 下記参照 |
| ペイウォールでの訴求 | 「あなたが"集中力リセット"を期待していると教えてくれました。専用のメッセージをご用意しました」 |

#### ゴール別メッセージセット

| ユーザーの期待（Q10回答） | 開始メッセージ | 終了メッセージ |
|--------------------------|--------------|--------------|
| 目の休憩 | 「目を休めましょう。画面はどこにも行きません」 | 「目がリフレッシュしました」 |
| 集中力リセット | 「今、正しいことをしていますか？」 | 「よし、次の一手を」 |
| 習慣化 | 「今日○回目の休憩です」 | 「また一歩、習慣に近づきました」 |
| ストレス軽減 | 「深く息を吸って...」 | 「少し軽くなりましたか？」 |
| 完全カスタム | ユーザーが自由に入力 | ユーザーが自由に入力 |

**なぜ刺さるか**: 診断で「自分が何を求めているか」を言語化した直後に、「それに合わせたメッセージ」を提示される。「このアプリは自分を理解している」という感覚。

---

### Pro機能④: スマートスケジュール（Smart Schedule）

**診断での伏線**: Q8「休憩を取れない理由: 切りが悪い」

| 項目 | 内容 |
|------|------|
| 概要 | 作業時間帯のみリマインド + おやすみ期間設定 |
| 無料版 | 常時リマインド（起動中は一律で通知） |
| Pro版 | 曜日×時間帯のスケジュール設定 / 「集中モード」連動 / カレンダー連携（会議中は自動スキップ） |
| ペイウォールでの訴求 | 「あなたは"切りが悪い"ことが休憩の障壁と答えました。Proなら会議中は自動でお休みします」 |

**なぜ刺さるか**: 「切りが悪い」と答えた人は、不適切なタイミングの通知にイラつく予備軍。7日間使って実際に「今じゃない」と思った経験がある。その課題をProが解決する。

---

### Pro機能⑤: 30日チャレンジ（Extended Challenge）

**診断での伏線**: Q11「いつ頃から始めたいですか？: 今すぐ」/ Day 7レポートの達成感

| 項目 | 内容 |
|------|------|
| 概要 | 7日間チャレンジの延長版。30日間の構造化されたプログラム |
| 無料版 | 7日間チャレンジ（オンボーディングの一部として無料提供） |
| Pro版 | 30日 / 90日チャレンジ。週ごとにテーマが変わり、段階的に閉眼時間を延長 |
| プログラム構成 | Week 1: 基礎（5秒固定）→ Week 2: 探索（3-10秒可変）→ Week 3: 深化（呼吸連動）→ Week 4: 自律（自分のリズム確立） |
| ペイウォールでの訴求 | 「7日間チャレンジの達成、おめでとうございます。次の30日で、習慣を"定着"させませんか？」 |

**なぜ刺さるか**: 7日間チャレンジを完了した直後は「もっと続けたい」モチベーションがピーク。無料では7日で終わる構造が、Proへの自然な導線になる。**7日間チャレンジ自体がProのフリートライアル**。

## Pro機能の価値マトリクス: 無料 vs Pro

| 領域 | 無料版 | Pro版 | 課金の「壁」となる心理 |
|------|--------|-------|---------------------|
| リマインド間隔 | 4段階の固定間隔 | データ駆動の自動調整 | 「固定だとタイミングが合わない」 |
| セッションメッセージ | 固定の1種類 | ゴール別セット + 完全カスタム | 「毎回同じ言葉で飽きた」 |
| 振り返り | 「今日○回」のみ | 週次レポート + ヒートマップ + 傾向分析 | 「自分の変化を見たい」 |
| チャレンジ | 7日間（オンボーディング） | 30日 / 90日の構造化プログラム | 「もっと続けたい」 |
| スケジュール | 常時 | 曜日×時間帯 + カレンダー連携 + 集中モード連動 | 「会議中に通知が来てうざい」 |
| ビジュアル | デフォルト | 目キャラの色・スタイル変更 | 「自分だけの相棒にしたい」 |
| サウンド | デフォルト音 | 複数の覚醒音から選択 | 「音を変えて気分を変えたい」 |

**上位3機能（適応リマインド・ウィークリーインサイト・カスタムメッセージ）が主力。**
下位4機能（チャレンジ・スケジュール・ビジュアル・サウンド）は追加の彩り。

## なぜこの組み合わせが刺さるか

### 1. 診断が「課題」を作り、Proが「解決」する構造

```
診断 Q5「何分おきに集中が切れる？」
  → ユーザー: 「15分」と回答
  → 知識: 「集中サイクルは人それぞれ」
  → 7日間: 固定30分間隔を体験
  → Day 7: 「あなたの実際のパターンは22分でした」
  → Pro: 「適応リマインドなら自動で22分に」
```

**診断で問題を自覚** → **無料版で「惜しい」体験** → **Proで完全な体験**

### 2. 7日間のデータが「証拠」になる

Day 1のペイウォールは「想像」で売る。Day 7のペイウォールは「実績」で売る。

- 「あなたは18回セッションしました」→ 事実
- 「火曜の午後にスキップ率が高い」→ 事実
- 「適応リマインドなら改善できます」→ 事実に基づく提案

**データに基づく提案は拒否しにくい。**

### 3. 無料版は「十分使える」が「もどかしい」

無料版でもBlindの本質的価値（5秒の閉眼）は完全に体験できる。
しかし7日間使うと:
- 「この間隔、自分に合ってない気がする」（→ 適応リマインド）
- 「先週と比べてどうなったか知りたい」（→ インサイト）
- 「毎回同じメッセージで飽きた」（→ カスタムメッセージ）

**使えば使うほど「もうちょっと」が出てくる設計。**

## リスクと対策

| リスク | 対策 |
|--------|------|
| 25画面で離脱率が高い | 診断パートにスキップ導線を用意。スキップ時は「おすすめプラン」で即開始 |
| Day 7まで到達しない | Day 3, Day 5でもライトな成果表示を挟む。小さな達成感を途中で提供 |
| 「押し売り感」でブランド毀損 | Day 1のペイウォールは極めてソフトに。Day 7も「無料で続ける」を明確に |
| macOSアプリとして長いオンボーディングが異質 | ノッチUIの中で展開するため、モバイルアプリのフルスクリーン感とは異なる。Blind独自の体験として成立させる |
| 適応リマインドの精度が低い | 最初は「あなたの平均セッション間隔は○分」→「この間隔をおすすめとして設定しますか？」の半自動で開始 |
