# Blind — 拡張オンボーディング設計（顧客育成型）

## 採用方針

**パターンA（自己診断型）をベースに、パターンB（7日間チャレンジ）のDay 7成果レポート+ペイウォールを組み合わせる。**

パターンCのストーリー型は不採用（効果が不確実）。

---

## ターゲットペルソナの再定義

### 「集中が足りない人」ではなく「過集中で暴走する人」

Blindのメインターゲットは**集中力が低い人ではない**。
むしろ**集中しすぎて、象（System 1 / 無意識の衝動）が暴走し、使い（理性 / メタ認知）が手綱を失う人**。

#### 象と象使いのメタファー

```
象使い（理性）: 「今日はAタスクが最重要だ」
象（衝動）:     「お、このバグ面白い。直そう。あ、リファクタもしよう。
                 ついでにCI設定も...」

── 3時間後 ──

象使い:         「...あれ、Aタスク一切手をつけてない」
```

**Blindの5秒は、暴走した象を止めて象使いに手綱を返す瞬間。**

#### このペルソナの特徴

| 特徴 | 集中力不足の人 | **過集中の人（Blindのターゲット）** |
|------|-------------|--------------------------------|
| 問題 | 集中できない | 集中しすぎて方向を見失う |
| 時間感覚 | 「やる気が出ない」 | 「気づいたら3時間経ってた」 |
| タスク管理 | そもそも着手できない | 着手したら止まれない。抜け漏れが出る |
| AI利用時 | AI使えば楽になる | AIが象を加速させる（ウサギの穴が深くなる） |
| 休憩の障壁 | やる気がない | **没頭していて気づかない / 中断が怖い** |
| 必要なもの | モチベーション | **強制的な中断 + メタ認知の瞬間** |
| Blindの価値 | リラックス | **「今、正しいことをしているか？」の確認** |

## Pro機能設計: 「暴走する象を止める」ための道具

### 設計原則

> **過集中ペルソナの課題から逆算する。「あったらいい」ではなく「象を止めるために必要」な機能だけ。**

従来のPro機能（見た目・音のカスタマイズ）はこのペルソナには刺さらない。
彼らが金を払うのは**「暴走を止めてくれる」確実性**に対して。

---

### Pro機能①: 暴走検知リマインド（Runaway Detection）

**診断での伏線**: Q5「最長何時間連続？」/ Q6「止めるまでどのくらいかかる？」

| 項目 | 内容 |
|------|------|
| 概要 | スキップ行動から「今、暴走中」を検知し、リマインドの強度を段階的に上げる |
| 無料版 | 固定間隔（15/30/45/60分）で均一に通知 |
| Pro版 | スキップすると次の通知が短い間隔で来る。2回連続スキップ → 半分の間隔に。3回連続 → さらに短く+音の変化 |
| 技術実装 | スキップ回数カウント + 間隔の動的調整。ローカル完結 |
| ペイウォールでの訴求 | 「あなたは7日間で○回連続スキップしました。それは象が暴走しているサインです。Proなら見逃しません」 |

**なぜ刺さるか**:
- Q9で「中断されるのがすごく嫌」と答えた人 = まさに暴走中に通知を無視する人
- 固定間隔は暴走中でも非暴走中でも同じ → **暴走中にこそ強く介入する**のが過集中ペルソナの本質的ニーズ
- 「象が暴走している時ほど、象使いの声が聞こえない」→ Proが声を大きくする

**段階的割り込みの設計**:
```
通常: 30分間隔の通知
  ↓ スキップ
15分後: やや強い通知（音の変化）
  ↓ またスキップ
7分後: 強い通知（画面にも表示）
  ↓ またスキップ
3分後: 最強通知（ノッチUIが自動展開）
```

### Pro機能②: 優先事項アンカー（Priority Anchor）

**診断での伏線**: Q4「予定と違う作業をしていた」/ Q7「本来のタスクが漏れた」

| 項目 | 内容 |
|------|------|
| 概要 | 1日の始めに「今日やるべきこと」を1-3個設定。各セッションの閉眼前に表示 |
| 無料版 | 固定メッセージ「目を閉じて、今を確かめよう」 |
| Pro版 | 「目を閉じる前に——今日の優先事項: ①Aタスク ②Bタスク ③Cタスク」→ 閉眼 → 開眼後「今やっていたことは、この中にありましたか？」 |
| 技術実装 | メニューバーから毎朝設定（3つまで）。セッションUIに表示 |
| ペイウォールでの訴求 | 「あなたは"優先順位を思い出したい"と答えました。Proなら毎回のセッションがチェックポイントになります」 |

**なぜ刺さるか**:
- 過集中ペルソナの最大の痛みは「頑張ったのに間違ったことをしていた」
- Blindの5秒が「目を休める時間」ではなく**「今日の優先事項を思い出す時間」**になる
- これが刺さるのは**自分で「暴走する」と自覚している人だけ**。診断がその自覚を作る
- **タスク管理アプリとの根本的な違い**: リストはある。問題はリストを見ていないこと。Blindは「リストを見ろ」と身体で割り込む

---

### Pro機能③: 暴走パターン分析（Runaway Insights）

**診断での伏線**: Q4「予定と違う作業をしていた」の頻度 / Day 7レポートの体験

| 項目 | 内容 |
|------|------|
| 概要 | いつ・どのくらい暴走しやすいかを可視化する週次レポート |
| 無料版 | メニューバーに「今日○回」のみ表示 |
| Pro版 | 週次レポート: 曜日×時間帯のスキップ率ヒートマップ / 連続スキップ発生回数 / 前週比較 |
| 技術実装 | セッションログ（時刻, 実行/スキップ, 連続スキップ数）をローカルDB保存 |
| ペイウォールでの訴求 | 「あなたは火曜の14-17時にスキップ率が80%——象が最も暴走しやすい時間帯です。Proなら毎週この傾向を追跡できます」 |

**なぜ刺さるか**:
- Day 7レポートで初めて「自分の暴走パターン」を可視化される
- **Day 7レポート自体がProのデモ**。「これが毎週見られる」は直感的に価値がわかる
- 過集中ペルソナは「パターンを知れば対策できる」と考えるタイプ（合理的・分析好き）
- 自分のデータを見るのが好きな層と親和性が高い

### Pro機能④: カスタム問いかけ（Custom Prompts）

**診断での伏線**: Q10「メタ認知の習慣は？」/ Q11「Blindに期待すること」

| 項目 | 内容 |
|------|------|
| 概要 | セッション開始時・終了時のメッセージを、過集中の文脈に合わせてカスタマイズ |
| 無料版 | 固定メッセージ（「目を閉じて、今を確かめよう」/「おかえり」） |
| Pro版 | ゴール別メッセージセット + 完全カスタム入力 |
| ペイウォールでの訴求 | 「あなたは"暴走を止めてほしい"と答えました。あなた専用の問いかけを用意しました」 |

#### ゴール別メッセージセット

| ユーザーの期待（Q11回答） | 開始メッセージ | 閉眼中の問い | 終了メッセージ |
|--------------------------|--------------|------------|--------------|
| 暴走を止めてほしい | 「一度、手を止めてください」 | 「今やっていることは、始めようと思ったことですか？」 | 「手綱を取り戻しましたか？」 |
| 優先順位を思い出したい | 「今日の最重要タスクは何でしたか？」 | 「それに向かっていますか？」 | 「よし、正しい山を登ろう」 |
| 定期的に立ち止まりたい | 「立ち止まる時間です」 | 「今の自分を俯瞰してみてください」 | 「また30分後に」 |
| 目を休めたい | 「目を休めましょう」 | — | 「お疲れさまでした」 |
| 完全カスタム | ユーザーが自由に入力 | ユーザーが自由に入力 | ユーザーが自由に入力 |

**なぜ刺さるか**:
- 過集中ペルソナにとって「目を閉じて、今を確かめよう」は漠然としすぎている
- **「今やっていることは、始めようと思ったことですか？」**は刺さる。まさにそれが暴走の核心
- 診断で自分のゴールを言語化した後だから、「自分に合ったメッセージ」の価値が実感できる

### Pro機能⑤: スマートスケジュール（Smart Schedule）

**診断での伏線**: Day 7レポートの暴走パターン

| 項目 | 内容 |
|------|------|
| 概要 | 暴走しやすい時間帯にリマインド頻度を自動で上げる |
| 無料版 | 常時均一の固定間隔 |
| Pro版 | 曜日×時間帯でリマインド間隔を変動。「火曜午後は20分間隔、金曜午前は45分間隔」 |
| 技術実装 | スキップ率の時間帯別集計 → 間隔の自動調整 |
| ペイウォールでの訴求 | 「あなたは午後に暴走しやすい。Proなら午後だけ間隔を短くできます」 |

**なぜ刺さるか**:
- 過集中ペルソナは「暴走しない時間帯」と「暴走する時間帯」の差が大きい
- 朝は冷静にタスクをこなせるが、午後にスイッチが入ると止まらない
- 均一な間隔は「朝はうるさい、午後は足りない」→ **時間帯別の強弱**が合理的

---

## Pro機能が刺さらないもの（不採用 or 優先度下げ）

| 機能 | なぜ過集中ペルソナに刺さらないか |
|------|-------------------------------|
| ビジュアルカスタマイズ（目キャラの色変更） | 暴走を止めてくれない。「かわいい」は購買動機にならない |
| サウンドカスタマイズ | 音の種類が増えても暴走は止まらない |
| 30日/90日チャレンジ | 過集中の人は「チャレンジ達成」より「今日の暴走を防ぐ」に関心。長期プログラムは離脱しやすい |
| カレンダー連携（会議中スキップ） | 過集中の人は会議中に暴走しない。暴走するのは一人で作業している時 |

**結論**: 過集中ペルソナが金を払うのは「見た目」でも「達成感」でもなく、**「暴走の検知と介入の精度」**。

## Pro機能の価値マトリクス: 無料 vs Pro

| 領域 | 無料版 | Pro版 | 過集中ペルソナの「壁」心理 |
|------|--------|-------|------------------------|
| リマインド | 固定間隔で均一 | スキップ連続で間隔短縮（暴走検知） | 「通知を無視した時こそ必要なのに」 |
| セッションメッセージ | 固定「目を閉じて、今を確かめよう」 | ゴール別 + カスタム問いかけ | 「"今を確かめよう"じゃ象は止まらない」 |
| 優先事項 | なし | 毎朝設定→毎セッション表示 | 「何をすべきか知ってる。忘れるのが問題」 |
| 振り返り | 「今日○回」のみ | 暴走パターン分析（ヒートマップ） | 「いつ暴走するか自分で把握したい」 |
| 時間帯調整 | 均一 | 暴走しやすい時間帯は高頻度 | 「午後が危ないのに朝と同じ間隔は無駄」 |

## なぜこの組み合わせが過集中ペルソナに刺さるか

### 1. 診断が「暴走の自覚」を作り、Proが「暴走の制御」を売る

```
診断 Q4「予定と違う作業をしていた？」→「毎日」と回答
  → 知識: 「それは象が暴走しているサインです」
  → 7日間: 固定間隔を体験。何度かスキップ（=暴走）
  → Day 7: 「あなたは火曜午後に暴走しやすい」
  → Pro: 「暴走検知リマインドなら、スキップするほど強く介入します」
```

### 2. 無料版は「象使い」、Proは「象の柵」

無料版のBlindは「30分ごとに声をかける象使い」。でも象が暴走したら声は聞こえない。
Proは「象が走り出したら柵が狭くなる仕組み」。スキップすればするほど介入が強くなる。

**過集中の人は「声をかけてくれるだけでは足りない」と7日間で実感する。**

### 3. 競合にこの切り口がない

| アプリ | 提供するもの | 過集中への対応 |
|--------|------------|-------------|
| ポモドーロ系 | 25分固定タイマー | 均一。暴走中でも同じ音 |
| Headspace | ガイド瞑想 | 瞑想する時間に過集中の人は来ない |
| Stretchly | ストレッチ通知 | 均一。無視したら次は同じ間隔後 |
| **Blind Pro** | **暴走検知 + 段階的割り込み** | **スキップするほど強く介入** |

**「無視したら強くなる」リマインダーは市場にない。**

## リスクと対策

| リスク | 対策 |
|--------|------|
| 「暴走検知」が鬱陶しく感じる | Pro設定で強度を3段階に調整可能にする。最初はマイルドから |

| Day 7まで到達しない | Day 3, Day 5でもライトな成果表示 |
| macOSアプリとして長いオンボーディングが異質 | ノッチUIの中で展開。モバイルのフルスクリーン感とは異なる体験 |
