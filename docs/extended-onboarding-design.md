# Blind — 拡張オンボーディング設計（顧客育成型）

## 採用方針

**パターンA（自己診断型）をベースに、パターンB（7日間チャレンジ）のDay 7成果レポート+ペイウォールを組み合わせる。**

パターンCのストーリー型は不採用（効果が不確実）。

---

## ターゲットペルソナの再定義

### 「集中が足りない人」ではなく「過集中で暴走する人」

Blindのメインターゲットは**集中力が低い人ではない**。
むしろ**集中しすぎて、象（System 1 / 無意識の衝動）が暴走し、使い（理性 / メタ認知）が手綱を失う人**。

#### 象と象使いのメタファー

```
象使い（理性）: 「今日はAタスクが最重要だ」
象（衝動）:     「お、このバグ面白い。直そう。あ、リファクタもしよう。
                 ついでにCI設定も...」

── 3時間後 ──

象使い:         「...あれ、Aタスク一切手をつけてない」
```

**Blindの5秒は、暴走した象を止めて象使いに手綱を返す瞬間。**

#### このペルソナの特徴

| 特徴 | 集中力不足の人 | **過集中の人（Blindのターゲット）** |
|------|-------------|--------------------------------|
| 問題 | 集中できない | 集中しすぎて方向を見失う |
| 時間感覚 | 「やる気が出ない」 | 「気づいたら3時間経ってた」 |
| タスク管理 | そもそも着手できない | 着手したら止まれない。抜け漏れが出る |
| AI利用時 | AI使えば楽になる | AIが象を加速させる（ウサギの穴が深くなる） |
| 休憩の障壁 | やる気がない | **没頭していて気づかない / 中断が怖い** |
| 必要なもの | モチベーション | **強制的な中断 + メタ認知の瞬間** |
| Blindの価値 | リラックス | **「今、正しいことをしているか？」の確認** |

#### 診断の全体設計意図

```
Block A（行動）: 何が起きているか → 「確かに暴走している」
     ↓
Block B（心理）: なぜ起きるのか → 「こういう思考パターンが原因かも」
     ↓
Block C（能力）: 自分で止められるか → 「気づいてるのに止められない。仕組みが要る」
     ↓
→ 知識教育: ここで初めて名前（内なる声、気づきの筋肉）を付けて体系化する
```

**診断では「聞く」だけ。教育は次のステップで行う。**
これにより「診断で自分の体験を語る → 知識で体系化される → "なるほど、そういうことか"」の流れが生まれる。

## Pro機能の全体設計: 5秒を「完全なトレーニングサイクル」に変える

### 無料版 vs Pro版のフレーミング

```
無料版 Blind:
  通知 → 目を閉じる（5秒）→ 目を開ける
  = 筋トレで言えば「ダンベルを1回持ち上げるだけ」

Pro版 Blind:
  通知 → ❶ 内なる声チェック → ❷ 目を閉じる（5秒）→ ❸ 象使いの判断 → ❹ 振り返り記録
  = 筋トレで言えば「フォーム指導 + 負荷調整 + 記録 + 進捗トラッキング付きのパーソナルジム」
```

**Proは「より良いリマインダー」ではなく「5秒間のメンタルフィットネスシステム」。**
同じ5秒でも、前後にガイドが入ることでトレーニング効果が倍増する。

### 5秒トレーニングサイクルの全体像

```
[通知] ──→ [❶ Pre-close] ──→ [❷ 閉眼 5秒] ──→ [❸ Post-close] ──→ [❹ 記録]
            Pro: 内なる声       無料/Pro共通     Pro: 象使いの      Pro: 1タップ
            チェック                              判断ガイド        ログ
```

---

### Pro機能①: Pre-closeチェック — 内なる声のラベリング（閉眼前 3秒）

**知識教育での伏線**: Step 2「内なる声の正体」/ 「声に気づくことが第一歩」

| 項目 | 内容 |
|------|------|
| 概要 | 目を閉じる直前に「今、どんな内なる声が聞こえていますか？」と1タップで確認 |
| 無料版 | 固定メッセージ「目を閉じて、今を確かめよう」→ すぐ閉眼 |
| Pro版 | 閉眼前に1タップ: 🏃もっと / 🔍完璧 / ✨好奇心 / 🎛️全部自分 / 😌なし（穏やか）|
| 所要時間 | 1-2秒（タップするだけ） |
| 技術実装 | セッション開始UIにタップボタン4+1個を追加 |

**なぜ刺さるか**:
- PQ理論の核心は「サボターに気づく（ラベリング）だけで力が弱まる」こと
- 名前を付ける行為自体がトレーニング。タップするだけで「あ、今"もっと派"が出てる」と気づく
- **オンボーディングで自分のタイプを知っているから**、1タップの選択が内省として機能する
- データが蓄積され、「いつどの声が出やすいか」がウィークリーレポートに反映される

---

### Pro機能②: Post-closeガイド — 象使いの判断（開眼後 5秒）

**知識教育での伏線**: Step 4「5秒の構造 ❸ 象使いが地図を確認する」

| 項目 | 内容 |
|------|------|
| 概要 | 目を開けた直後に「次に何をすべきか」を象使い視点で判断させるガイド |
| 無料版 | 「おかえり」→ セッション終了 |
| Pro版 | 開眼後、ユーザーの状況に応じた問いかけ → 1タップ応答 |
| 所要時間 | 3-5秒 |
| 技術実装 | セッション終了UIに問いかけ + 選択肢を追加 |

**Post-closeの問いかけパターン**:

| 条件 | 問いかけ | 選択肢 |
|------|---------|--------|
| 優先事項が設定済み | 「今やっていたことは、今日の優先事項に含まれていますか？」 | はい / いいえ、戻ろう / 意図的に別のことをしている |
| 内なる声チェックで「もっと」を選択 | 「"もっとやれる"の声が聞こえていましたね。今の作業は、本当に今やるべきことですか？」 | はい、正しい / いいえ、方向転換する |
| 内なる声チェックで「完璧」を選択 | 「今の作業は80%で十分ではありませんか？」 | まだ必要 / 確かに、次に行こう |
| 内なる声チェックで「好奇心」を選択 | 「面白い寄り道でしたか？元のタスクに戻りますか？」 | 戻る / もう少し（意図的） |
| 内なる声チェックで「穏やか」を選択 | 「いいペースですね。このまま続けましょう」 | （自動終了） |

**なぜ刺さるか**:
- 閉眼で「中断」は起きた。でもその後「何をすべきか」のガイドがないと、また象が走り出す
- **Pre-closeで特定した内なる声のタイプに応じて問いかけが変わる** → パーソナライズされた介入
- 「はい / いいえ」の1タップが**象使いの意思決定**そのもの。判断を強制することがトレーニング
- 「いいえ、方向転換する」を選んだ瞬間が、**ユーザーにとって最も価値を感じる瞬間**

---

### Pro機能③: トレーニングログ — 気づきの筋力トラッキング

**知識教育での伏線**: Step 3「気づきの力は筋肉。使うほど強くなる」

| 項目 | 内容 |
|------|------|
| 概要 | Pre-close / Post-closeのデータを蓄積し、「気づきの筋力」を可視化 |
| 無料版 | メニューバーに「今日○回」のみ |
| Pro版 | ウィークリーレポート（下記参照） |
| 技術実装 | セッションログ（時刻, 内なる声タイプ, Post-close判断, スキップ有無）をローカルDB保存 |

**ウィークリーレポートの内容**:

| セクション | 内容 | 教育的意味 |
|-----------|------|----------|
| 気づきスコア | 「今週、○回セッションのうち○回で内なる声に気づけました（○%）」 | 自己認知の定量化 |
| 内なる声の分布 | 🏃32% / 🔍45% / ✨18% / 🎛️5% 「あなたは"完璧派"の声が最も多い週でした」 | どのパターンが支配的か可視化 |
| 暴走パターン | 曜日×時間帯のスキップ率ヒートマップ + 内なる声タイプの紐付け | いつ・どの声が暴走を起こすか |
| 軌道修正回数 | 「Post-closeで"方向転換する"を選んだ回数: ○回」「この○回が、間違った山を登り続けるのを防ぎました」 | 自己制御の定量化。**これがBlindの最大の価値** |
| 成長トレンド | 前週比: 気づきスコア↑ / スキップ率↓ / 軌道修正↑ | 長期的な成長の可視化 |

**なぜ刺さるか**:
- Day 7レポートがそのままこの機能のデモ。「これが毎週見られる」は直感的
- **「軌道修正回数」が最強のメトリクス**: 「今週3回、間違った方向に進むのを防ぎました」は具体的な価値
- 過集中ペルソナは分析好き。自分のデータを見るのが好き
- 「気づきの筋肉が○%成長」は筋トレメタファーの延長。成長実感がモチベーションになる

### Pro機能④: 暴走検知リマインド（Runaway Detection）

**知識教育での伏線**: Step 3「気づいても止まれない。仕組みがないだけ」

| 項目 | 内容 |
|------|------|
| 概要 | スキップ行動から「今、暴走中」を検知し、リマインドの強度を段階的に上げる |
| 無料版 | 固定間隔（15/30/45/60分）で均一に通知 |
| Pro版 | スキップすると次の通知が短い間隔に。2回連続スキップ → 半分の間隔。3回連続 → さらに短く+音の変化 |
| 技術実装 | スキップ回数カウント + 間隔の動的調整。ローカル完結 |

**段階的割り込みの設計**:
```
通常: 30分間隔の通知
  ↓ スキップ（内なる声が活動開始）
15分後: やや強い通知（音の変化）
  ↓ またスキップ（内なる声が本格稼働）
7分後: 強い通知（画面にも表示）
  ↓ またスキップ（完全に暴走）
3分後: 最強通知（ノッチUIが自動展開）
```

**なぜ刺さるか**: Q13で「わかっていてもやめられない」と答えた人に直撃。「自分にない自己制御力を外部に持ってもらう」装置。

---

### Pro機能⑤: 優先事項アンカー（Priority Anchor）

**知識教育での伏線**: Step 4「❸ 象使いが地図を確認する」

| 項目 | 内容 |
|------|------|
| 概要 | 1日の始めに「今日やるべきこと」を1-3個設定。Post-closeガイドで毎回表示 |
| 無料版 | 固定メッセージ「目を閉じて、今を確かめよう」 |
| Pro版 | 「今日の優先事項: ①○○ ②○○ ③○○」→ 閉眼 → 開眼後「今やっていたことは、この中にありましたか？」 |
| 技術実装 | メニューバーから毎朝設定（3つまで）。Post-closeガイドに統合表示 |

---

### Pro機能⑥: スマートスケジュール（Smart Schedule）

| 項目 | 内容 |
|------|------|
| 概要 | 暴走しやすい時間帯にリマインド頻度を自動で上げる |
| 無料版 | 常時均一の固定間隔 |
| Pro版 | トレーニングログのデータから、曜日×時間帯でリマインド間隔を自動変動 |

## Pro機能が刺さらないもの（不採用 or 優先度下げ）

| 機能 | なぜ刺さらないか |
|------|----------------|
| ビジュアルカスタマイズ | 気づきの筋肉を鍛えてくれない。見た目の変化は購買動機にならない |
| サウンドカスタマイズ | 音の種類が増えてもトレーニング効果は変わらない |
| 30日/90日チャレンジ | 過集中の人は「今日の暴走を防ぐ」に関心。長期プログラムは離脱しやすい |
| カレンダー連携 | 暴走するのは一人で作業している時。会議中は問題ない |

## Pro機能の価値マトリクス: 無料 vs Pro

| 領域 | 無料版 | Pro版 | 心理的な「壁」 |
|------|--------|-------|--------------|
| 5秒の体験 | 目を閉じて開ける | ❶内なる声チェック→❷閉眼→❸象使い判断→❹記録 | 「ただ閉じるだけじゃ変わらない気がする」 |
| セッションメッセージ | 固定「今を確かめよう」 | 内なる声タイプ別の問いかけ | 「毎回同じ言葉じゃ気づきが深まらない」 |
| 振り返り | 「今日○回」のみ | 気づきスコア + 内なる声分布 + 軌道修正回数 + 成長トレンド | 「本当にトレーニングになってるか知りたい」 |
| リマインド | 固定間隔で均一 | スキップ連続で間隔短縮 | 「無視した時こそ必要なのに」 |
| 優先事項 | なし | 毎朝設定→毎セッション表示 | 「何をすべきか忘れるのが問題」 |
| 時間帯調整 | 均一 | 暴走しやすい時間帯は高頻度 | 「午後が危ないのに朝と同じ間隔は無駄」 |

## なぜこの設計が刺さるか

### 1. 教育が「自分ごと」になる構造

```
一般的なアプリ:  理論を説明 → 「へー」 → 忘れる
Blind:          体験を質問で語らせる → 理論で名前を付ける → 「あ、それ自分だ」→ 忘れない
```

**診断で先に体験を語らせているから、後の教育が「他人事」にならない。**

### 2. 「5秒のリマインダー」→「5秒のパーソナルジム」

```
無料版: 通知 → 5秒閉眼 → 終わり（ダンベルを1回上げただけ）
Pro版:  通知 → 内なる声チェック → 5秒閉眼 → 象使い判断 → 記録（完全なワークアウト）
```

**同じ5秒でも、前後にガイドが入ることで「トレーニング」として認知される。**
ユーザーが得る価値が「中断」から「成長」に変わる。

### 3. 「気づきの筋肉」メタファーがPro課金を正当化する

```
「筋トレの器具は無料で貸します（無料版）。
 でもフォーム指導、負荷調整、成長トラッキングは
 パーソナルトレーナーについてもらう必要があります（Pro版）」
```

**課金の理由が「機能を買う」ではなく「自分の成長に投資する」になる。**
これはNoomやHeadspaceが証明している心理モデル。

### 4. 競合にこの切り口がない

| アプリ | 提供するもの | 5秒間に起きること |
|--------|------------|-----------------|
| ポモドーロ系 | 25分固定タイマー | タイマーが鳴るだけ |
| Headspace | ガイド瞑想 | 10-60分の瞑想（長すぎて過集中の人は来ない） |
| Stretchly | ストレッチ通知 | ストレッチ指示。メタ認知なし |
| **Blind Pro** | **5秒メンタルフィットネス** | **内なる声ラベリング → 閉眼 → 象使い判断 → 記録** |

## リスクと対策

| リスク | 対策 |
|--------|------|
| 33画面で離脱率が高い | 各ブロック冒頭にスキップ導線。Block Aのみでも最低限のプラン生成が可能 |
| Pre-close/Post-closeが面倒になる | 各1-2タップ・3-5秒で完結する設計。面倒なら非表示にできるトグル |
| 「内なる声」「気づきの筋肉」がピンとこない | 診断で先に体験を語らせているので「それ自分だ」となりやすい。ならない場合はスキップ可 |
| 5秒+前後のガイドでトータル15秒になる | Pre-close/Post-closeは任意。5秒の閉眼だけでも完結する設計を維持 |
| macOSアプリとして長いオンボーディングが異質 | ノッチUIの中で展開。モバイルのフルスクリーン感とは異なる体験として設計 |
| Day 7まで到達しない | Day 3, Day 5でもライトな成果表示で興味を維持 |
