# Blind — サービス体験設計書

## 1. サービス概要

Blindは、macOSのメニューバーに常駐するマインドフルネスリマインダーアプリ。

**コアコンセプト**: 「目を閉じる」という身体的アクションを通じて、意図的な中断を作り、「今、正しいことをしているか？」を確認する機会を提供する。

**ブランド**: コミカルでかわいい。MacBookのノッチに住む目のキャラクターが、ユーザーに休憩を促す。テクノロジーが身体性を取り戻すための道具になる。

**ひとことで**: 「ノッチに住む目が、あなたの目を閉じさせてくれるアプリ」

---

## 2. ターゲットユーザー

### プライマリ: AI時代の知識労働者（25-40歳）

- AIエージェントを並列で走らせながら、次の指示を考え、結果をレビューし、また次を投げる。思考が途切れない
- AIの進化速度についていこうとして、新しいモデル・ツール・ワークフローのキャッチアップが終わらない
- 「AIが考えてくれる分、自分の頭が鈍くなっている気がする」という漠然とした不安
- AIに委譲できるタスクが増えるほど、「自分は今、何をすべきか」の判断が難しくなる
- 生産性ツールは好きだが、仰々しいマインドフルネスアプリには抵抗がある
- 「瞑想」は少しハードルが高いが、「目を閉じる」なら5秒でできる

**原体験**: このサービスは、開発者自身がAIのビッグウェイブに乗り切れず疲弊していた時に「目を閉じる」という手法に出会い、5秒の中断が思考をリセットしてくれることに気づいたことから生まれた。AIと共に働く人にこそ、意図的な「中断」が必要。

### セカンダリ: クリエイター・フリーランス

- 自宅で一人で作業。休憩のリズムを自分で作る必要がある
- ツールの見た目やブランドにこだわる。ダサいアプリは使わない
- MacBookの「ノッチに住むキャラクター」というコンセプトに共感しやすい

### アンチペルソナ（ターゲットではない）

- 瞑想の深い実践者（もっと本格的なアプリを使う）
- Windows/Linuxユーザー（macOS専用）
- 作業中の通知を一切受けたくない人

## 3. ターゲットシーン

### シーン1: AIエージェントの合間に

> 16:00。Claude Codeに3つのタスクを並列で投げて、結果を待ちながら次の指示を考えている。
> Slackの未読、GitHubの通知、新しいモデルのリリースブログ。情報が止まらない。
> ノッチから目がキョロキョロと現れる。「あ、Blindだ」。
> 5秒間目を閉じる。暗闇の中で気づく。
> 「今やってる3つのタスク、そもそも優先度が間違ってる」。
> 目を開けて、2つをキャンセルし、本当に重要な1つに集中し直した。
> AIが考えてくれる時代だからこそ、「何を考えさせるか」を決める5秒が要る。

### シーン2: 朝の最初の一息

> 9:00。Macを開いてSlackを確認しようとした瞬間、ノッチから目が出現。
> ログイン時起動を設定していたので、最初の休憩リマインドが来た。
> 5秒間目を閉じて、「今日の一番大事なタスクは何だろう」と考える。
> 目を開けて、Slackではなくタスクリストを開いた。

### シーン3: 「AIに頭を持っていかれた」後のリセット

> 14:30。AIとの壁打ちを2時間続けて、大量のアウトプットが出た。
> でも頭がぼんやりしている。AIの出力を追いかけているうちに、自分の思考が消えた気がする。
> Blindのリマインドが来る。5秒間目を閉じる。
> 「待って、このプロジェクトで自分が本当にやりたかったことは何だっけ」。
> 目を開けて、AIの提案を半分捨て、自分の直感を信じた方向に舵を切った。

## 4. サービスジャーニー

### Phase 0: 認知・導入

| ステップ | ユーザー行動 | タッチポイント |
|---------|------------|--------------|
| 発見 | SNS/口コミで知る | 「ノッチに目が住んでる」スクショがバズる |
| 興味 | GitHubページを見る | README + スクショ + GIF |
| 導入 | DMGをダウンロード | GitHub Releases |
| インストール | DMGからApplicationsへドラッグ | macOS標準フロー |

**WoW**: 「ノッチに目が住んでる」という一目でわかるビジュアルフック。スクショだけで何のアプリか伝わる。

---

### Phase 1: オンボーディング（初回起動）

| ステップ | 画面 | 目キャラ状態 | テキスト |
|---------|------|-----------|--------|
| Welcome | ノッチ + 拡張テキスト帯 | `.idle`（キョロキョロ） | 「はじめまして！」「Blindは、画面から離れる小さな休憩を作るアプリです」 |
| Camera | ノッチ + 拡張テキスト帯 | `.searching`（左右を見回す） | 「カメラを使います」「目を閉じたことを感知するために使います。映像は保存・送信しません。」 |
| TrySession | 通常セッション（2秒短縮版） | `.tracking` → `.closing` | 「目を閉じて、今を確かめよう」 |
| Done | ノッチ + 拡張テキスト帯 | `.winking` | 「準備完了！」「30分ごとにお知らせします。設定はメニューバーからいつでも。」 |

**WoW**: 説明を読むのではなく、実際に「目を閉じる」体験を最速（2秒）で提供。「ああ、こういうことか」と身体で理解できる。バックドロップ・音量制御なしで安心感を優先。

**設計意図**:
- 目キャラが先に現れ、ユーザーとの「出会い」を演出する
- カメラ権限は「なぜ必要か」を説明してから要求（Apple HIG推奨パターン）
- 拒否しても完了できる（強制しない）
- 体験ファースト: 文章を読ませるのではなく、やってみせる

---

### Phase 2: 日常利用（リマインドループ）

```
[作業中] ──30分経過──→ [通知] ──タップ──→ [セッション] ──完了──→ [作業に戻る]
                         │                                           │
                         └──────────────────── 30分 ────────────────┘
```

#### 2-1. リマインド通知

macOS標準通知で「休憩しませんか？」と表示。通知をタップするとセッション開始。

**通知メッセージのランダム性**: 同じ文言だとスルーされやすい。100パターン程度の通知文言を用意し、ランダムに表示する。利用回数に応じて出現率を調整し、飽きを防ぐ。

**狙い**: 通知は「指示」ではなく「提案」。無視しても罪悪感がないトーン。毎回少しだけ違う言葉で、「今日は何て言うんだろう」という小さな楽しみを作る。

#### 2-2. セッション（4フェーズ）

| Phase | 名前 | 時間 | 演出 | ユーザー体験 |
|-------|------|------|------|------------|
| 1 | Summon（召喚） | ~0.6s | ノッチから黒UIがにゅっと伸びる | 「お、来た」 |
| 2 | Encounter（邂逅） | ユーザー依存 | 目キャラがユーザーの顔を認識、テキスト表示 | 「目を閉じればいいんだな」 |
| 3 | Immersion（没入） | 5秒（設定可） | 画面全体が黒に。音量フェードダウン | 「静かだ...今日の午後、何をすべきだろう」 |
| 4 | Awakening（覚醒） | ~2.0s | 黒が縮小。音量復帰。目キャラがウインク | 「おかえり。よし、次はあれをやろう」 |

**WoW**:
- **Summon**: ノッチという「ハードウェアの一部」からキャラクターが出現する驚き。アプリではなく、Macの一部のように感じる
- **Encounter**: 目キャラがユーザーの顔を認識してロックオンする瞬間。「見られている」感覚がマインドフルネスの入口になる
- **Immersion**: 画面全体が暗転し、音も小さくなる。デジタルデバイスが「閉じる」体験を提供するという逆説。5秒間の完全な静寂
- **Awakening**: 「おかえり」の一言。大げさな称賛ではなく、さりげない迎え。ウインクのユーモア

#### 2-3. セッション中の安全設計

| 手段 | 動作 |
|------|------|
| ×ボタン | テキスト帯に常時表示。タップで即座に終了 |
| ESCキー | いつでもセッション中断 |
| Watchdog | メインスレッド10秒ハング → 自動復帰 |
| タイムアウト | 120秒で強制終了 |
| 音量保護 | 最低0.02（完全無音にしない）。クラッシュ時も復帰 |

**狙い**: 「画面が真っ暗になって操作できなくなるかも」という不安を6層の防御で消す。安心して目を閉じられる環境を作る。

---

### Phase 3: 習慣化（1週間〜）

| 指標 | 目標 |
|------|------|
| Day 1 | オンボーディング完了。初めて目を閉じる |
| Day 3 | 通知に反応してセッションを2-3回実行 |
| Day 7 | 「ログイン時に起動」を有効化 |
| Day 14 | リマインド間隔を自分好みに調整 |
| Day 30 | Blindなしの日に違和感を覚える |

**習慣化の仕掛け**:
- 低コスト: 1回5秒。「やらない理由」がない
- 即時報酬: 目を開けた瞬間の「リフレッシュ感」が報酬そのもの
- 愛着: 目キャラのウインクが小さな楽しみになる
- 押し付けない: 通知を無視してもペナルティなし。罪悪感ゼロ

## 5. オンボーディング詳細

→ [ADR-0003: 初回オンボーディングフロー](adr/0003-onboarding-flow.md)

### オンボーディングのゴール

1. **5秒で体験**: 「目を閉じる」アクションを身体で理解する
2. **カメラ許可**: 自然な文脈で権限を取得する
3. **期待値設定**: 30分ごとの通知があることを伝える
4. **愛着形成**: 目キャラとの「出会い」を印象的にする

### オンボーディングのKPI

| 指標 | 目標 |
|------|------|
| オンボーディング完了率 | 80%以上 |
| カメラ権限許可率 | 70%以上 |
| お試しセッション体験率 | 60%以上（カメラ許可者の85%） |
| 初日セッション実行率 | 50%以上（通知からの初回セッション） |

## 6. 設定項目

### 現在（MVP）

| 設定 | 選択肢 | デフォルト | 意図 |
|------|--------|----------|------|
| リマインド間隔 | 15 / 30 / 45 / 60分 | 30分 | ポモドーロ(25分)より長く、忘れない程度 |
| 目を閉じる時間 | 3 / 5 / 10秒 | 5秒 | 短すぎず長すぎない。呼吸1回分 |
| サウンド | カスタム選択 | デフォルト音 | 覚醒時の完了音。macOS通知音と被らない独自音源 |
| ログイン時に起動 | ON / OFF | OFF | 習慣化した後にONにしてもらう |
| チュートリアルを再表示 | ボタン | — | いつでも再体験可能 |

#### 追加候補
- 時間間隔ではなく時間指定でのリマインド（例：12時、17時など複数設定）

### 設計方針

- **選択肢は最小限**: 「カスタマイズ疲れ」を避ける。3-4個の選択肢で十分
- **デフォルトが最適解**: 設定を変えなくても良い体験が得られる
- **段階的な開放**: MVPでは4項目。有料版で拡張

### 将来（Pro版）

| 設定 | 概要 | 狙い |
|------|------|------|
| ビジュアルカスタマイズ | 目キャラの色・スタイル | 自分だけの相棒感 |
| サウンドカスタマイズ | 複数の覚醒音（旧Mac起動音風なども） | 体験のバリエーション。macOS通知音と被らない独自性 |
| スケジュール | 作業時間帯のみリマインド | 週末は休み |
| スヌーズ | 「5分後に再通知」 | 今は無理、でも忘れたくない |

#### 追加候補
- 開始前と開始後のワードをカスタマイズできる機能。何に気をつけたいか、どう出迎えて欲しいかユーザーが決められる
- CLIコマンド（`blind://start-session` URL schemeの拡張）。AIエージェントが発火タイミングを制御できるようにし、タイマーだけでなく文脈に応じたリマインドを可能にする

## 7. 体験設計の狙い

### 7-1. 「マインドフルネス」を脱構築する

従来のマインドフルネスアプリは「瞑想」「呼吸法」「ガイド音声」を提供する。Blindはそのどれもやらない。

**やること**: 目を閉じる。5秒間。以上。

これは「マインドフルネス」を名乗らない。「休憩」とすら言わない。ただ「目を閉じて、今を確かめよう」と言うだけ。行為の解釈はユーザーに委ねる。

**狙い**: ハードルを極限まで下げることで、「マインドフルネスなんて自分には関係ない」と思っている人にリーチする。

### 7-2. ハードウェアとソフトウェアの境界を溶かす

ノッチは「邪魔な黒い部分」として認識されている。Blindはそこにキャラクターを住まわせることで、ハードウェアの欠点をソフトウェアの個性に変える。

**狙い**: 「アプリを使っている」ではなく「Macの一部が話しかけてきた」という感覚。インストール後、ノッチを見るたびにBlindを思い出す。ハードウェアがマーケティングチャネルになる。

### 7-3. 身体性の回復

スクリーンタイムは「目を開けて画面を見る」行為の連続。Blindは唯一「目を閉じる」ことを求めるアプリ。

デジタルデバイスが「画面を見るな」と言う逆説。画面が真っ暗になり、音が消える。5秒間だけ、デジタルから解放される。しかしその体験を提供しているのもデジタルデバイスである。

**狙い**: テクノロジーが身体性を取り戻すための道具になれることを証明する。

### 7-4. 愛着と継続

目キャラのデザインは「コミカルでかわいい」。仰々しくない。禅の教えではなく、ノッチから覗くちょっとおとぼけな目。

ウインク、キョロキョロ、まばたき。これらの小さなアニメーションが「また会いたい」を作る。

**狙い**: 機能ではなくキャラクターで継続させる。Blindを「使う」のではなく、目キャラに「会いに行く」感覚。

### 7-5. 5秒の問い

目を閉じている5秒間、ユーザーの脳は自動的に現状を振り返る。

- 「さっきからTwitterを30分見てた」
- 「このデザイン、方向性が違うかも」
- 「肩が凝ってる。ストレッチしよう」
- 「今日の午後、一番大事なタスクは何だろう」

Blindは何も言わない。ただ暗闇を提供するだけ。問いはユーザー自身が生み出す。

**狙い**: 外部からの「指示」ではなく、内側からの「気づき」。これがBlindの本質的な価値。

## 8. 3モード対応

| モード | 対象デバイス | 形状 | 位置 |
|--------|------------|------|------|
| `.notch` | MacBook Pro 2021+ | 上辺フラット + 凹角丸 | 物理ノッチと融合 |
| `.noNotch` | iMac, Mac mini, 外部ディスプレイ | 上辺フラット + 凹角丸 | 画面上部中央にフェイクノッチ |
| `.island` | 将来のカメラアイランドMac | ピル形状（全角丸） | アイランド下に拡張 |

**狙い**: どのMacでも同じ体験。ノッチがないMacでも「ノッチっぽいもの」が出現し、ブランドの一貫性を保つ。物理ノッチがあるMacではハードウェアと融合し、「Macの一部」感が最大化される。

## 付録: 競合との差別化

| 項目 | Blind | Headspace | Stretchly | Time Out |
|------|-------|-----------|-----------|----------|
| アクション | 目を閉じる（5秒） | 瞑想（5-60分） | ストレッチ | 休憩（画面ロック） |
| キャラクター | ノッチの目 | なし | なし | なし |
| 身体検知 | カメラ（目の状態） | なし | なし | なし |
| 所要時間 | 5秒 | 5-60分 | 30秒-5分 | 1-5分 |
| ハードル | 極低 | 高 | 中 | 中 |
| 価格 | 無料 + Pro買い切り | サブスク | 無料 | 有料 |
| プラットフォーム | macOS | 全プラットフォーム | 全プラットフォーム | macOS |

**Blindの独自性**: 「身体を使う」×「キャラクター」×「5秒」の組み合わせ。どの競合とも重ならない。

## 付録: AIエージェント連携構想

ターゲットユーザーがAI時代の知識労働者である以上、BlindもAIワークフローに溶け込むべき。

### CLI / URL Scheme

```bash
# URL schemeでセッション開始（既存）
open "blind://start-session"

# 将来: CLIコマンド
blind start          # セッション開始
blind status         # 状態確認（idle / in-session）
blind config         # 設定のJSON出力
```

**ユースケース**:
- Claude Codeの長時間タスク完了後に自動でBlindセッションを挿入
- AIエージェントが「ユーザーが2時間連続作業している」と判断してトリガー
- Pomodoro的なワークフローをAIが動的に組み立てる

**狙い**: タイマーという「固定間隔」から、AIが文脈を読む「適応的リマインド」へ。人間が設定するのではなく、AIが最適なタイミングを選ぶ。
